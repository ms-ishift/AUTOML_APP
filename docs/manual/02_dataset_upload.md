## 데이터 업로드

선택한 프로젝트에 **CSV / XLSX** 파일을 업로드해 프로파일(컬럼별 타입 ·
결측 · 유니크 비율) 과 미리보기를 자동 생성합니다.

### 👤 사용자 관점

**할 수 있는 일**

- 파일 업로드 (드래그앤드롭 지원)
- 자동 프로파일 확인 — 타입, 결측 비율, 유니크 개수
- 상위 N 행 미리보기 (기본 50 행)
- 업로드 이력 확인 · 불필요한 데이터셋 삭제

**기본 사용 순서**

1. 좌측 상단 **파일 업로드** 영역에 CSV/XLSX 를 드롭 또는 선택.
2. 업로드 완료 후 **프로파일** 탭에서 컬럼별 통계를 확인.
3. **미리보기** 탭에서 상위 행을 확인하고 문제 없으면 다음 단계(학습) 로 이동.

**자주 쓰는 버튼/옵션**

| 버튼/위젯 | 설명 |
| --- | --- |
| 파일 선택/드롭 | CSV/XLSX 1건. 크기 제한은 `.env` 의 `MAX_UPLOAD_MB` (기본 200MB, `FR-030`). |
| 미리보기 행 수 | 기본 50, 최대 500. 성능 보호 상한선 (`FR-032`). |
| 삭제 | 연결된 학습/모델이 없을 때만 제거 가능 (`FR-035`). |

**주의사항 & 팁**

- 헤더(첫 행) 가 없는 파일은 거부됩니다 — pandas 가 자동 "Unnamed" 로 치환한 컬럼은 `HEADER_MISSING` 오류로 귀결됩니다.
- **중복 컬럼명** 은 업로드 단계에서 명시적으로 차단 (`DUPLICATED_COLUMNS`).
- 인코딩은 UTF-8 권장. CP949 등은 자동 추정을 시도하지만 실패 시 에러 안내가 뜹니다.

### 🛠 개발자 관점

| 항목 | 위치 |
| --- | --- |
| 페이지 | `pages/02_dataset_upload.py` |
| 서비스 | `services/dataset_service.py` (`upload_dataset`, `get_dataset_profile`, `preview_dataset`, `delete_dataset`) |
| 유틸 | `utils/file_utils.py` (`parse_tabular`, `validate_columns`) |
| ML | `ml/profiling.py` (타입 추정 / 결측 / 유니크 프로파일) |

| FR | FR-030 ~ FR-035 |

**확장 포인트**

- 여러 파일 동시 업로드 / 자동 병합 — 현재는 1 건씩만.
- 타입 수동 오버라이드 UI — 현재는 프로파일러 자동 추정 결과를 그대로 사용.
- 대용량 스트리밍 업로드 (> 1GB) — `MAX_UPLOAD_MB` 상향 + 청크 파서 필요.

### 자주 겪는 오류

| 메시지 | 원인 | 복구 |
| --- | --- | --- |
| `파일이 비어 있습니다.` | 0 byte 업로드 | 다른 파일 선택 |
| `깨진 CSV` / 파싱 오류 | 인코딩 불일치 또는 구분자 비정형 | UTF-8 로 다시 저장하거나 구분자 점검 |
| `중복된 컬럼명` | 같은 헤더가 둘 이상 | 원본에서 헤더를 유일하게 정리 |
| `헤더가 비어 있거나 Unnamed 컬럼 존재` | 첫 행이 값 아님 | 올바른 헤더 행으로 재작성 |
| `파일이 MAX_UPLOAD_MB 를 초과` | 환경 한도 초과 | `.env` 에서 `MAX_UPLOAD_MB` 수정 → `make run` (자동 동기화) |
