## 예측

저장된 모델에 대해 **단건 폼** 또는 **파일(CSV)** 로 예측을 수행하고,
결과를 화면 확인 · CSV 다운로드로 받습니다.

### 👤 사용자 관점

**할 수 있는 일**

- 단건 예측 — 모델 스키마에 따라 입력 폼이 자동 생성됩니다 (`FR-082`).
- 파일 예측 — CSV 를 업로드하면 필수 컬럼 검증 후 일괄 예측 (`FR-081`).
- 결과 확률/점수 표시 (분류의 경우 클래스 확률 포함, `FR-084`).
- 결과 CSV 다운로드 — 원본 + `prediction` 컬럼 (`FR-085`).

**기본 사용 순서**

1. 대상 **모델** 을 선택합니다. 모델 관리에서 "예측으로 이동" 을 거쳐오면 자동 선택됩니다.
2. 탭에서 **단건** 또는 **파일** 을 고릅니다.
3. 단건: 입력값을 폼에 채우고 *예측*. 파일: CSV 업로드 후 *예측*.
4. 결과를 확인하고 *CSV 다운로드* 로 저장합니다.

**자주 쓰는 버튼/옵션**

| 위젯 | 설명 |
| --- | --- |
| 모델 선택 | 현재 프로젝트의 저장 모델 (`FR-080`) |
| 파일 업로드 | CSV 만 허용. 모델 `schema.json` 의 필수 컬럼이 누락되면 거부 (`FR-083`) |
| 예측 | 실행. 실패 시 Service 예외가 카드에 렌더됩니다 (`FR-084`) |
| CSV 다운로드 | 입력 컬럼 + 예측 결과 (+ 확률) 를 합친 CSV (`FR-085`) |

**주의사항 & 팁**

- 파일 예측에서 **컬럼 이름이 학습 시점과 정확히 일치** 해야 합니다. 대소문자·공백 차이도 오류입니다.
- 문자열 범주형은 학습 시 보았던 값 외엔 자동으로 `Unknown` 처리되며 정확도가 낮아질 수 있습니다.
- 분류 결과는 *예측 클래스* + *상위 확률* 컬럼이 붙고, 회귀는 *예측 값* 단일 컬럼이 붙습니다.

**확장 포인트**

- 비동기 배치 예측 — 대용량(>10MB) CSV 에 대한 진행률/재시도. 현재는 동기 수행.
- 외부 API 엔드포인트 노출 — `prediction_service.predict_batch` 를 FastAPI 라우트로 감싸면 됨.
- A/B 테스트: `predict_single` 인자에 `candidate_model_id` 추가 후 비교 로그.

### 자주 겪는 오류

| 메시지 | 원인 | 복구 |
| --- | --- | --- |
| `누락된 입력 컬럼: [...]` | CSV 헤더가 학습 스키마와 다름 | 원본 CSV 헤더를 모델 스키마에 맞춰 수정 |
| `값 형식 오류 (...)` | 숫자 컬럼에 문자열 등 | 단건 폼은 입력 제한, 파일은 사전 정제 후 재시도 |
| 결과 CSV 크기 0 | 업로드 CSV 가 비어 있거나 전부 유효성 실패 | 업로드 파일을 점검하고 필수 컬럼 확인 |
