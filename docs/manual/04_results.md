## 결과 비교

최근(혹은 선택한) 학습의 **알고리즘별 성능 표 · 시각화 · 베스트 추천** 을
확인하고, 원하는 알고리즘을 ★ 모델로 저장합니다.

### 👤 사용자 관점

**할 수 있는 일**

- 알고리즘별 지표 표 · 실행 시간 · 실패 여부 확인 (`FR-070`)
- 베스트 모델 자동 추천 — 기준 지표 방향(↑/↓) 에 따라 정해집니다 (`FR-071`)
- 시각화: 분류는 혼동 행렬, 회귀는 실제 vs 예측 산점도 (`FR-072`)
- ★ 버튼으로 **모델 저장** → 예측 페이지에서 바로 사용 가능 (`FR-073`)

**기본 사용 순서**

1. 학습 페이지에서 실행이 끝나면 자동으로 이 페이지로 넘어옵니다.
2. 표에서 지표가 가장 좋은 행(기본 정렬) 을 확인합니다.
3. 원하는 행의 **★ 저장** 버튼을 눌러 모델을 영구화합니다.
4. 시각화 탭으로 모델 품질을 눈으로 점검합니다.

**자주 쓰는 버튼/옵션**

| 버튼 | 설명 |
| --- | --- |
| ★ 저장 | 해당 알고리즘 결과를 `models/<model_id>/` 에 아티팩트로 저장합니다. |
| 이전 학습 선택 | 드롭다운으로 과거 job 을 불러옵니다. |
| 예측으로 이동 | 저장 직후 곧바로 예측 페이지로 `switch_page`. |

**주의사항 & 팁**

- 지표 방향(↑ 클수록 좋음 / ↓ 작을수록 좋음) 은 지표 옆에 표시됩니다.
- 실패한 알고리즘은 표에 남아 있지만 점수는 공란(NaN) 입니다 — ★ 저장 대상이 아닙니다.
- 같은 학습 결과를 여러 번 저장해도 **다른 model_id** 로 중복 저장됩니다 (버전 관리는 §FR-074 참조).

### 🛠 개발자 관점

| 항목 | 위치 |
| --- | --- |
| 페이지 | `pages/04_results.py` |
| 서비스 | `services/training_service.get_training_result` · `services/model_service.save_model` |
| 시각화 | 페이지 내부에서 `plotly.express` (혼동행렬 heatmap, 산점도) |
| DTO | `services/dto.TrainingResultDTO` · `AlgoResultDTO` |
| FR | FR-070 ~ FR-074 |

**확장 포인트**

- 시각화 추가(ROC 곡선, feature importance) — `plotly` 컴포넌트만 추가하면 됨.
- 비교표 컬럼(추가 지표, cross-val 분산) 커스터마이즈 — `AlgoResultDTO` 확장 + 표 렌더 수정.
- 저장 전 승인 워크플로(§8.3) — 모델 상태 enum 을 `candidate/approved/archived` 로 분리.

### 자주 겪는 오류

| 메시지 | 원인 | 복구 |
| --- | --- | --- |
| `표시할 학습 결과가 없습니다.` | 해당 프로젝트의 학습이 0건 | 학습 페이지에서 먼저 실행 |
| `★ 저장 실패` | 아티팩트 저장 경로 권한/디스크 문제 | `storage/models/` 디렉터리 권한·용량 확인 |
| 시각화 빈 화면 | 실패한 알고리즘을 선택한 경우 | 다른 성공 알고리즘 선택 |
