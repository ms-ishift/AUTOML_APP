## 프로젝트 관리

데이터·학습·모델을 묶는 **작업 공간** 단위입니다. 앱의 모든 기능은 현재
선택된 프로젝트 컨텍스트에서 동작하므로, 가장 먼저 방문하는 페이지입니다.

### 👤 사용자 관점

**할 수 있는 일**

- 프로젝트 생성 / 이름·설명 수정 / 삭제
- 카드를 눌러 **현재 프로젝트** 로 선택 (사이드바에 반영)
- 데이터셋 수 / 저장 모델 수 / 업데이트 시각을 카드에서 한눈에 확인

**기본 사용 순서**

1. 상단 *"새 프로젝트"* 영역에 이름을 입력하고 "생성".
2. 목록 카드의 **선택** 버튼을 눌러 작업 대상을 정합니다.
3. 카드 *수정* 버튼으로 이름/설명을 바꾸거나, *삭제* 로 제거합니다.

**자주 쓰는 버튼/옵션**

| 버튼 | 설명 |
| --- | --- |
| 생성 | 새 프로젝트를 추가하고 곧바로 선택합니다 (`FR-020`). |
| 선택 | 해당 프로젝트를 현재 작업 컨텍스트로 지정합니다 (`FR-022`). |
| 수정 | 이름·설명을 갱신합니다 (`FR-023`). |
| 삭제 | **데이터·모델·학습 이력이 없을 때만** 삭제 가능합니다 (`FR-024`). |

**주의사항 & 팁**

- 프로젝트 이름은 **중복 불가** 입니다. 같은 이름을 쓰려면 먼저 이전 프로젝트를 삭제하거나 수정하세요.
- 연결된 데이터셋·모델이 있는 프로젝트는 삭제가 거부됩니다 — 관련 자원을 먼저 정리해야 합니다.

**확장 포인트**

- 대량 프로젝트 대응: 카드 렌더링을 페이지네이션으로 전환.
- 권한 분기: 현재는 단일 시스템 사용자(`SYSTEM_USER_ID=0`) — §8.1 인증 확장 시 `owner_user_id` 기준 필터 추가.

### 자주 겪는 오류

| 메시지 | 원인 | 복구 |
| --- | --- | --- |
| `이미 존재하는 프로젝트 이름입니다.` | 중복 이름으로 생성 시도 | 이름 변경 또는 기존 프로젝트 수정/삭제 |
| `연결된 데이터셋/모델이 있어 삭제할 수 없습니다.` | 관련 자원이 남아 있음 | 데이터 업로드·모델 관리 페이지에서 먼저 정리 |
| 카드가 비어 있음 / `프로젝트가 없습니다` | 초기 상태 | 상단 "새 프로젝트" 로 1건 생성 |
