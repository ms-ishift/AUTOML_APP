## 전체 흐름 개요

정형 데이터(CSV/XLSX) 한 장을 올려 놓으면 **업로드 → 프로파일 → 학습 →
비교 → 저장 → 예측** 까지 한 흐름으로 진행합니다. 현재 선택된 프로젝트는
사이드바에서 언제든 확인/해제할 수 있습니다.

```
[홈] → [프로젝트] → [데이터 업로드] → [학습] → [결과 비교]
                                           │
                                           ▼
                               [모델 관리] ─▶ [예측]
                                           │
                                           ▼
                                     [이력/관리자]
```

### 👤 빠른 사용 순서

1. **프로젝트** 에서 새 프로젝트를 만들고 선택합니다.
2. **데이터 업로드** 에서 CSV/XLSX 를 올립니다.
3. **학습** 에서 타깃 컬럼과 기준 지표를 고른 뒤 "학습 실행".
   - 필요하면 하단 *"고급 전처리 (선택)"* 에서 스케일·인코딩·datetime 분해·불균형 대응을 조정하고, *"미리보기"* 로 변환 결과를 확인할 수 있습니다 (`FR-055~058`).
4. **결과 비교** 에서 ★ 베스트 모델을 저장합니다.
5. **예측** 에서 단건 폼 또는 파일(CSV) 로 결과를 확인하고 CSV 로 내려받습니다.

> 처음이라면 `samples/classification.csv` 파일을 업로드해 보세요.
> 타깃 컬럼은 `species`, 지표는 `accuracy` 가 기본 예시입니다.

### 🛠 아키텍처 포인트

| 레이어 | 역할 | 대표 파일 |
| --- | --- | --- |
| UI | Streamlit 페이지·컴포넌트 | `app.py`, `pages/`, `pages/components/` |
| Service | DTO 기반 오케스트레이션 | `services/*_service.py` |
| ML | 순수 함수(레지스트리·전처리·학습·평가) | `ml/` |
| Repository | SQLAlchemy ORM | `repositories/` |

세부 설계는 [`ARCHITECTURE.md`](../../ARCHITECTURE.md), 진행 체크리스트는
[`IMPLEMENTATION_PLAN.md`](../../IMPLEMENTATION_PLAN.md), 요구사항 원문은
[`AutoML_Streamlit_MVP.md`](../../AutoML_Streamlit_MVP.md) 에서 확인합니다.

### 용어 미니사전

| 용어 | 뜻 |
| --- | --- |
| 프로젝트 | 데이터·학습·모델을 묶는 최상위 컨테이너 (`FR-020`) |
| 데이터셋 | 프로젝트에 업로드된 단일 CSV/XLSX (`FR-030`) |
| 학습(Job) | 하나의 설정으로 여러 알고리즘을 동시 학습한 실행 단위 (`FR-060`) |
| 모델 | 학습 결과 중 "저장" 된 단일 알고리즘 아티팩트 (`FR-073`) |
| 예측(Job) | 저장된 모델로 수행한 단건/파일 예측 실행 단위 (`FR-080`) |
